我们的承诺

游导旅游致力于为所有用户提供平等、便捷的访问体验。我们承诺遵循 《Web内容无障碍指南 (WCAG) 2.1》 AA级标准，确保我们的网站对所有用户都是可访问的，包括但不限于：

  • 使用屏幕阅读器的视障用户
  • 使用键盘导航的运动障碍用户
  • 患有认知障碍或学习障碍的用户
  • 老年用户
  • 使用移动设备的用户
  • 使用低带宽连接的用户

✅ 已实现的无障碍特性

🔍 清晰的焦点指示

所有可交互元素都具有清晰可见的键盘焦点样式，确保使用键盘导航的用户能够清楚知道自己当前所在位置。

⌨️ 完整的键盘支持

网站所有功能均可通过键盘操作完成，包括导航、表单填写、按钮点击等，无需使用鼠标。

📱 响应式设计

采用移动优先的响应式设计，确保网站在各种设备和屏幕尺寸上都能良好显示和操作。

🎨 足够的颜色对比度

文字与背景的颜色对比度符合 WCAG AA 标准（至少 4.5:1），确保内容易于阅读。

📝 语义化HTML

使用正确的HTML语义标签（如 nav, main, header, footer, article 等），便于辅助技术正确解析页面结构。

🖼️ 图片替代文本

所有有意义的图片都配有描述性的 alt 文本，装饰性图片有适当的空 alt 属性。

🔗 链接文本描述

所有链接都有清晰描述性的文本，避免使用"点击这里"等无意义描述。

📺 ARIA 标签

使用 ARIA 属性增强动态内容和复杂组件的可访问性。

⏱️ 动画控制

尊重用户的动画偏好设置，减少或移除不必要的动画效果。

🗺️ 跳转链接

提供跳转链接，让用户可以快速跳转到主要内容或导航区域。

📐 表单标签与错误提示

所有表单字段都有清晰的标签和必填标记，错误信息明确指出问题所在。

🌓 深色模式

支持系统级深色模式，并提供手动切换选项。

📋 WCAG 2.1 AA 标准合规详情

1. 可感知性 (Perceivable)

  • 1.1.1 非文本内容 - 所有非文本内容都有替代文本
  • 1.3.1 信息和关系 - 使用语义化HTML传递信息和结构
  • 1.4.1 颜色的使用 - 颜色不是唯一的信息传达方式
  • 1.4.3 对比度（最小） - 文本对比度至少 4.5:1
  • 1.4.4 文本缩放 - 支持高达 200% 的文本缩放

2. 可操作性 (Operable)

  • 2.1.1 键盘 - 所有功能可通过键盘操作
  • 2.1.2 无键盘陷阱 - 焦点可以自由移动
  • 2.4.1 绕过块 - 提供跳转链接
  • 2.4.2 页面标题 - 每个页面都有描述性标题
  • 2.4.3 焦点顺序 - 焦点顺序合乎逻辑
  • 2.4.4 链接目的 - 链接文本描述清晰
  • 2.4.7 焦点可见 - 键盘焦点可见

3. 可理解性 (Understandable)

  • 3.1.1 页面语言 - HTML lang 属性正确设置
  • 3.2.1 焦点变化 - 焦点变化不会自动触发重大变化
  • 3.3.1 错误识别 - 表单错误清晰识别
  • 3.3.2 标签或说明 - 表单有清晰的标签和说明

4. 健壮性 (Robust)

  • 4.1.1 语法分析 - HTML 语法正确有效
  • 4.1.2 名称、角色、值 - 组件有正确的名称、角色和状态

🛠️ 可访问性工具

我们提供了多种可访问性辅助功能，您可以根据需要进行调整：

  • 高对比度模式 - 增强文字与背景的对比度
  • 字体大小调节 - 调整页面文字大小
  • 深色模式 - 切换深色/浅色主题
  • 动画减弱 - 减少或关闭页面动画

⚠️ 已知限制

我们正在持续努力改进网站的无障碍性能。以下是当前已知的限制：

  • 部分第三方嵌入内容（如地图、视频播放器）可能尚未完全符合无障碍标准
  • 某些历史遗留内容可能尚未完全优化
  • PDF 文档可能需要额外处理以提供最佳的可访问性

我们正在积极解决这些问题，并持续更新网站以提高可访问性。
