📋 缓存策略说明

1 强缓存 (Cache-Control) 

Cache-Control: max-age=31536000, public
  • • 适用于静态资源：CSS、JS、图片、字体
  • • 缓存时间：1年（建议用于版本化文件）
  • • 用户再次访问时无需请求服务器

2 协商缓存 (ETag/Last-Modified) 

ETag: "33a64df551425fcc55e4d42a148795d9"
Last-Modified: Wed, 21 Oct 2015 07:28:00 GMT
  • • 适用于频繁更新的资源
  • • 服务器验证资源是否变更
  • • 304响应节省带宽

3 Service Worker 缓存 

self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
  );
});
  • • 支持离线访问
  • • 细粒度缓存控制
  • • 后台同步能力

4 IndexedDB 本地存储 

const db = await openDB('travel', 1, {
  upgrade(db) {
    db.createObjectStore('cache');
  }
});
  • • 存储结构化数据
  • • 大容量存储
  • • 支持索引查询

💡 缓存优化建议

🎯 最佳实践

  • 1. 为静态资源使用长缓存 + 版本化文件名
  • 2. 使用 CDN 加速资源分发
  • 3. 合理设置 Cache-Control 头
  • 4. 定期清理过期缓存

📊 预期收益

  • • 重复访问加载时间减少 50-80%
  • • 降低服务器带宽成本
  • • 提升用户体验满意度
  • • 减少网络请求数量
